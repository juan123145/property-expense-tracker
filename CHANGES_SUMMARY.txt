SHARED PROPERTY TRANSACTIONS - DEBUG & FIX SUMMARY
================================================================================

ISSUE
=====
User B shares Property 1 with User A, but User A only sees their own 
transactions, not User B's transactions in shared properties.

ROOT CAUSE
==========
The application has TWO parallel permission systems that are OUT OF SYNC:
  1. OLD: propertyShares table (being phased out)
  2. NEW: propertyMemberships table (replacement)

When properties are accessed:
  - getAccessiblePropertyIds() checks propertyShares (OLD)
  - canReadProperty() checks propertyMemberships (NEW)
  
Result: Properties appear accessible (OLD check passes) but transaction 
access is denied (NEW check fails).

SOLUTION IMPLEMENTED
====================
HYBRID APPROACH: Both systems now checked in parallel

CHANGED FILES
=============

1. lib/auth-utils.ts
   - Added: import propertyMemberships
   - Modified: getAccessiblePropertyIds() to query both systems
   - Behavior: Returns properties from propertyShares AND propertyMemberships
   - Impact: No breaking changes, backward compatible

2. lib/permissions.ts  
   - Added: import propertyShares
   - Modified: getUserPropertyRole() to check propertyShares fallback
   - Modified: canReadProperty() to check propertyShares fallback
   - Behavior: Tries propertyMemberships first, falls back to propertyShares
   - Impact: No breaking changes, supports legacy shares
